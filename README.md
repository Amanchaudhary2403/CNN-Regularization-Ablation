## тЧИ SYSTEM INIT тАФ The Question
 
Can BatchNorm, Dropout, Data Augmentation, and L2 Regularization тАФ individually and combined тАФ actually **improve generalization** when you're severely data-constrained?
 
We trained **6 controlled CNN variants** on just 5,000 CIFAR-10 images (10% of the full dataset) to find out. Each variant isolates one or more regularization choices while holding the base architecture constant. The results are surprising.

## ЁЯТб What We Learned

### 1 ┬╖ BatchNorm accelerates overfitting when used alone

BN on 5K samples means computing statistics from batches that cover only **0.64% of the data**. Running estimates are noisy and biased. BN speeds up convergence тАФ but with no explicit regularization, faster convergence just means **faster memorization**.

`M2` hit **100% training accuracy** by epoch 90. Validation never crossed 65.6%.

---

### 2 ┬╖ SpatialDropout + BatchNorm is the best combination

These two techniques complement each other:

- **SpatialDropout** drops *entire feature maps*, not individual units. It prevents the network from relying on any single spatial feature тАФ breaking feature co-adaptation at the channel level.
- **BatchNorm** then re-stabilizes the surviving activations, preventing gradient collapse or dead ReLUs caused by the aggressive dropout.

`M4` тАФ the only variant with both тАФ achieves **70.5% test accuracy** with just a **12.3% overfit gap**. No other variant comes close to both metrics simultaneously.

---

### 3 ┬╖ Data augmentation *hurt* performance on 32├Ч32 images

`V5` added `RandomRotation(0.05)`, `RandomZoom(0.05)`, and `RandomContrast(0.1)` on top of M4. It scored **68.7%** тАФ *1.8% lower* than M4 with no augmentation.

Why? CIFAR-10 images are tiny. Discriminative features (an ear, a wheel, a wing) occupy **10тАУ20 pixels**. A 5% zoom crops 2 pixels off each edge. An 18┬░ rotation smears fine details into neighbouring pixels. The model ends up training on distorted artefacts that simply don't exist in the test set.

Augmentation creates invariance only when there's enough resolution to survive the transform.

---

### 4 ┬╖ Stack too many constraints and the network can't learn anything

`M6` applies dropout (0.5) + L2 (1e-3) + BN + augmentation simultaneously. Training accuracy: **46.6%**. The model is below chance on several classes. It can't even fit the training data.

Each constraint reduces capacity. When constraints compound, you don't get a more robust model тАФ you get a model with **no remaining degrees of freedom** to learn from.

## ЁЯЦ╝я╕П Augmentation on Low-Resolution Data

| Transform | Effect on 32├Ч32 CIFAR | Use? |
|:---|:---|:---:|
| `RandomFlip('horizontal')` | Mirrors object naturally, structure preserved | тЬЕ Safe |
| `RandomRotation(0.05)` | ┬▒18┬░ smears 10тАУ20px features into noise | тЭМ Avoid |
| `RandomZoom(0.05)` | Crops to 30├Ч30, loses boundary context | тЭМ Avoid |
| `RandomContrast(0.1)` | Shifts colour distributions that BN already handles | тЭМ Avoid |
| Cutout / RandomErasing | Masks critical signal on already-small objects | тЭМ Avoid |
| Mixup / CutMix | Highly effective at full scale; harmful under 10K | тЭМ Avoid |

**Recommended augmentation for `< 64├Ч64`, `< 10K` samples:**

```python
augmentation = keras.Sequential([
    keras.layers.RandomFlip("horizontal")   # the only safe choice
])
```

## ЁЯУР Engineering Rules

Derived directly from experimental evidence тАФ not intuition.

```
R1  тЖТ  SpatialDropout2D (0.2тАУ0.3) + BatchNorm together, not separately
R2  тЖТ  No geometric augmentation on images < 64├Ч64 pixels
R3  тЖТ  BatchNorm goes between Conv and ReLU (pre-activation placement)
R4  тЖТ  SpatialDropout2D in conv blocks, standard Dropout in dense blocks only
R5  тЖТ  Total regularization budget < 0.6 (normalized); M6 hit 1.0 and collapsed
R6  тЖТ  Early stopping on val_loss, not val_accuracy (loss detects overconfidence earlier)
R7  тЖТ  BatchNorm alone is not a regularizer тАФ pair it with Dropout or L2 on small data
```

## ЁЯТб What We Learned

### 1 ┬╖ BatchNorm accelerates overfitting when used alone

BN on 5K samples means computing statistics from batches that cover only **0.64% of the data**. Running estimates are noisy and biased. BN speeds up convergence тАФ but with no explicit regularization, faster convergence just means **faster memorization**.

`M2` hit **100% training accuracy** by epoch 90. Validation never crossed 65.6%.

---

### 2 ┬╖ SpatialDropout + BatchNorm is the best combination

These two techniques complement each other:

- **SpatialDropout** drops *entire feature maps*, not individual units. It prevents the network from relying on any single spatial feature тАФ breaking feature co-adaptation at the channel level.
- **BatchNorm** then re-stabilizes the surviving activations, preventing gradient collapse or dead ReLUs caused by the aggressive dropout.

`M4` тАФ the only variant with both тАФ achieves **70.5% test accuracy** with just a **12.3% overfit gap**. No other variant comes close to both metrics simultaneously.

---

### 3 ┬╖ Data augmentation *hurt* performance on 32├Ч32 images

`V5` added `RandomRotation(0.05)`, `RandomZoom(0.05)`, and `RandomContrast(0.1)` on top of M4. It scored **68.7%** тАФ *1.8% lower* than M4 with no augmentation.

Why? CIFAR-10 images are tiny. Discriminative features (an ear, a wheel, a wing) occupy **10тАУ20 pixels**. A 5% zoom crops 2 pixels off each edge. An 18┬░ rotation smears fine details into neighboring pixels. The model ends up training on distorted artifacts that simply don't exist in the test set.

Augmentation creates invariance only when there's enough resolution to survive the transform.

---

### 4 ┬╖ Stack too many constraints and the network can't learn anything

`M6` applies dropout (0.5) + L2 (1e-3) + BN + augmentation simultaneously. Training accuracy: **46.6%**. The model is below chance on several classes. It can't even fit the training data.

Each constraint reduces capacity. When constraints compound, you don't get a more robust model тАФ you get a model that has **no remaining degrees of freedom** to learn from.

## ЁЯЦ╝я╕П Augmentation on Low-Resolution Data

| Transform | Effect on 32├Ч32 CIFAR | Use? |
|:---|:---|:---:|
| `RandomFlip('horizontal')` | Mirrors object naturally, structure preserved | тЬЕ Safe |
| `RandomRotation(0.05)` | ┬▒18┬░ smears 10тАУ20px features into noise | тЭМ Avoid |
| `RandomZoom(0.05)` | Crops to 30├Ч30, loses boundary context | тЭМ Avoid |
| `RandomContrast(0.1)` | Shifts color distributions that BN already handles | тЭМ Avoid |
| Cutout / RandomErasing | Masks critical signal on already-small objects | тЭМ Avoid |
| Mixup / CutMix | Highly effective at full scale; harmful under 10K | тЭМ Avoid |

**Recommended augmentation for `< 64├Ч64`, `< 10K` samples:**

```python
augmentation = keras.Sequential([
    keras.layers.RandomFlip("horizontal")   # the only safe choice
])
```

## ЁЯУР Engineering Rules

Derived directly from experimental evidence тАФ not intuition.

```
R1  тЖТ  SpatialDropout2D (0.2тАУ0.3) + BatchNorm together, not separately
R2  тЖТ  No geometric augmentation on images < 64├Ч64 pixels
R3  тЖТ  BatchNorm goes between Conv and ReLU (pre-activation placement)
R4  тЖТ  SpatialDropout2D in conv blocks, standard Dropout in dense blocks only
R5  тЖТ  Total regularization budget < 0.6 (normalized); M6 hit 1.0 and collapsed
R6  тЖТ  Early stopping on val_loss, not val_accuracy (loss detects overconfidence earlier)
R7  тЖТ  BatchNorm alone is not a regularizer тАФ pair it with Dropout or L2 on small data
```
